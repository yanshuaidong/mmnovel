## 命名规范

### 章节文件

格式：`{三位序号}-{短标题}.md`

示例：`001-序幕.md`、`012-夜访.md`

数字前缀保证排序；改标题时不必改序号。

### 章节 front matter

每章文件顶部建议添加：

```yaml
---
title: 第一章 序幕
status: draft          # draft | revised | final
pov: 视角人物
word_target: 3000      # 目标字数（可选）
---
```

## 创作流程

```
素材 → 策划 → 草稿 → 正文 → 分析
```

1. **素材** — 收集参考资料与灵感
2. **策划** — 完善大纲、人物、世界观
3. **草稿** — 试写与探索
4. **正文** — 章节定稿，更新 `正文/目录.md`
5. **分析** — 通读改稿，记录于 `策划/分析/`
